01 — Overview

Project Overview & Objectives

Phishing is the #1 attack vector used in over 80% of all reported security incidents globally. It exploits human psychology rather than technical vulnerabilities — meaning no firewall, no EDR, and no SIEM can stop a user who clicks a convincing malicious link.

This project runs a safe, controlled phishing simulation using GoPhish and Mailtrap, measures real susceptibility rates across a consenting test group, and produces a professional security awareness report with concrete, prioritised remediation recommendations — exactly what a security consultant delivers to clients after a phishing assessment engagement.

02 — Background

What Is Phishing? Why It Matters

Phishing is social engineering via email — crafting a message that appears to come from a trusted source and manipulating the recipient into taking an action: clicking a link, downloading a file, or entering credentials.

TypeDescriptionThis Project
PhishingMass emails to many recipients✅ Simulated
Spear PhishingTargeted at named individuals✅ Template approach
VishingVoice/phone social engineeringOut of scope
SmishingSMS-based phishingOut of scope
WhalingExecutives and high-value targetsReferenced in report
MITRE ATT&CK Mapping
This project simulates T1566.001 (Spearphishing Link) and T1566.002 (Spearphishing Attachment) — both under the Initial Access tactic. Including this mapping elevates your awareness report to professional threat-intelligence quality.

05 — Mailtrap Setup

Set Up Mailtrap (Safe Email Relay)

Mailtrap is a fake SMTP server — GoPhish sends emails to it, but they never reach real inboxes. Instead Mailtrap traps them in a virtual inbox so you can preview exactly how the email renders.

1
Create a free Mailtrap account
Go to https://mailtrap.io → Sign Up Free → verify email. No credit card needed.
2
Get SMTP credentials
Email Testing → Inboxes → My Inbox → SMTP Settings → select GoPhish from the integration dropdown. Copy Host, Port, Username, Password.
3
Create Sending Profile in GoPhish
GoPhish → Sending Profiles → New Profile:
Name: Mailtrap Test Relay
SMTP From: it-security@yourcompany.com
Host: sandbox.smtp.mailtrap.io:2525
Username & Password: from Mailtrap dashboard
4
Send a test email and confirm receipt
Click Send Test Email → enter your own email → check the Mailtrap inbox. The email should appear within 5 seconds.
Expected Result
Test email appears in Mailtrap inbox. You can inspect HTML source, raw headers, spam score, and preview the email exactly as a recipient would see it. ✅
06 — Email Template

Design the Phishing Email Template

This template mimics a corporate IT password reset notice — the most effective phishing pretext. It uses five psychological triggers: urgency, authority, fear, familiarity, and scarcity.

6.1 Psychological Triggers Used

TriggerHow UsedWhy It Works
Urgency"Account locked in 24 hours"Bypasses critical thinking — people act fast without analysing
AuthorityAppears from IT Security deptRecipients trust authority figures and comply without questioning
Fear"Unusual sign-in activity detected"Emotional response overrides logical analysis
FamiliarityCompany branding, logo, coloursFamiliar visual identity reduces suspicion significantly
Scarcity"One-time link expires in 24 hours"Creates pressure to act immediately before analysing

07 — Landing Page

Build the Credential Harvesting Landing Page

When recipients click the link, they land on a fake IT portal login page that mimics the company's real login portal. GoPhish captures every form submission — recording that the recipient entered credentials, without actually storing real passwords for any legitimate use.

⚠ No real credentials are stored dangerously
GoPhish records submission events for reporting purposes only. All captured test data is deleted at campaign close. The purpose is to measure susceptibility — not to harvest real credentials.
1
Go to Landing Pages in GoPhish
GoPhish → Landing Pages → New Page → Name: IT Portal — Password Verification
2
Paste the HTML login form
Use GoPhish's HTML editor to create a convincing IT portal login page. The form must have name="username" and name="password" fields for GoPhish to capture them.
3
Enable credential capture
Tick "Capture Submitted Data" and "Capture Passwords". Set Redirect URL to the real company login page (so victim doesn't get suspicious after submitting).
4
Preview and save
Click Preview to confirm the page renders correctly in browser. Save when satisfied.

10 — Results Analysis

Analyse Campaign Results with Python

#!/usr/bin/env python3 — Phishing Campaign Analyser
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('campaign_results.csv')
df['time'] = pd.to_datetime(df['time'])

total_sent      = len(df[df['message'] == 'Email Sent'])
total_opened    = len(df[df['message'] == 'Email Opened'])
total_clicked   = len(df[df['message'] == 'Clicked Link'])
total_submitted = len(df[df['message'] == 'Submitted Data'])

print(f"Sent:      {total_sent}")
print(f"Opened:    {total_opened} ({total_opened/total_sent*100:.1f}%)")
print(f"Clicked:   {total_clicked} ({total_clicked/total_sent*100:.1f}%)")
print(f"Submitted: {total_submitted} ({total_submitted/total_sent*100:.1f}%)")

# Segment by role
click_events = df[df['message'] == 'Clicked Link']
role_clicks = click_events.groupby('position').size()
print("\nClick rate by role:")
print(role_clicks)

11 — Indicator Analysis

Phishing Indicator Analysis — 8 Red Flags

#Red FlagWhat to Look ForSeverity
1Sender domain mismatchDisplay name says "IT Security" but domain is yourcompany-alerts.net not yourcompany.comCritical
2Urgency language"24 hours", "suspended", "immediately" — designed to prevent careful analysisHigh
3Fear-based trigger"Unusual sign-in activity" with no specific device, location or time detailsHigh
4Suspicious link URLHovering over the button reveals an IP address, not the company domainCritical
5HTTP not HTTPS on landing pageBrowser shows "Not Secure" — real IT portals always use HTTPSCritical
6Request for password via email linkLegitimate companies never ask you to enter your password by clicking an email linkCritical
7No personalisationReal IT alerts include last login time, device name, location, and employee IDMedium
8No digital signatureChecking email headers shows SPF and DKIM failures — it's not from a legitimate serverHigh

Correct Response When Suspicious

1
Hover over any link before clicking
Check the URL in the status bar. If it doesn't match the company's official domain exactly — do not click.
2
Expand the sender's actual email address
Click the sender display name to reveal the full address. Any domain mismatch is an immediate red flag.
3
Contact IT directly — not through the email
Call or message IT through a known, verified channel (Teams, Slack, direct phone). Never reply to the suspicious email.
4
Report using the "Report Phishing" button
Use the Outlook/Gmail plugin or forward to security@yourcompany.com. This is the correct and expected response.
