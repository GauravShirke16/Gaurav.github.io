Phishing Simulation & Awareness Report

Status: Planned Social Engineering Awareness Training Tools: GoPhish · Mailtrap
This project is planned and will be documented here once completed. All simulations will be conducted in a controlled, consensual test environment only.
Overview

Phishing is the #1 attack vector used by cybercriminals — responsible for over 80% of security incidents. Understanding how phishing works from the attacker's perspective is essential for any security engineer designing defences.

This project uses GoPhish (a free, open-source phishing framework) to run a safe, controlled simulation — sending fake phishing emails to a test group (myself and willing participants) and measuring the results. The output is a professional awareness report with real metrics and remediation recommendations.

Step-by-step methodology
01
Set up GoPhish
Download GoPhish and run it locally. Configure an SMTP relay using Mailtrap (a free, safe email testing service that never delivers to real inboxes).
02
Design a realistic phishing template
Create a convincing email template mimicking an IT department password reset notice. Study real phishing indicators: urgency language, spoofed domains, deceptive links.
03
Create a credential harvesting landing page
Build a fake login page using GoPhish's template editor. Configure it to capture clicks and form submissions without storing any real credentials.
04
Launch the campaign
Send the test campaign to a defined group of consenting participants. Track opens, clicks, and submissions in GoPhish's real-time dashboard.
05
Analyse results
Export campaign data. Analyse click rates by time of day, email subject line effectiveness, and which user profiles are most susceptible.
06
Write the awareness report
Produce a professional PDF report covering: campaign methodology, results, risk rating, remediation recommendations, and a proposed security awareness training plan.
