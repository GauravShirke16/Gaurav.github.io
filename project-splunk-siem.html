Splunk SIEM Dashboard

Status: Planned Threat Detection Log Analysis SIEM Tools: Splunk · SPL · Sysmon
This project is planned and will be documented here once completed. The structure below outlines the full methodology I will follow.
Overview

A SIEM (Security Information and Event Management) system is the command centre of any security operations team. It collects logs from every device on the network and uses rules and queries to surface suspicious patterns that humans would miss.

This project uses Splunk Free (up to 500MB/day) to build a functional security dashboard. Since I already use Splunk professionally at Essen Vision, this project deepens that expertise and produces documented evidence of my skills.

Step-by-step methodology
01
Install Splunk Free
Download and install Splunk Enterprise free tier on a Windows or Linux VM. Register for a free developer licence — no credit card needed.
02
Install Sysmon for rich Windows logs
Deploy Sysmon (System Monitor) on a Windows VM with the SwiftOnSecurity config. This captures process creation, network connections, file modifications — far richer than default Windows logs.
03
Forward logs to Splunk
Configure Splunk Universal Forwarder on endpoints. Set up inputs.conf to collect Windows Event Logs, Sysmon, and firewall logs.
04
Simulate attack scenarios
From the Kali Linux home lab, run a brute-force attack, a port scan, and a lateral movement simulation. These generate real attack telemetry in Splunk.
05
Write SPL detection queries
Use Splunk Processing Language (SPL) to write searches that detect failed logins, suspicious process executions, and unusual outbound connections.
06
Build dashboards and alerts
Create a SOC-style dashboard with panels for top failed logins, login by geography, process execution timeline, and a real-time alert for critical events.
Sample SPL detection query — brute force
# Detect brute force: 5+ failed logins in 5 minutes from same source index=windows EventCode=4625 | bucket _time span=5m | stats count by _time, src_ip, user | where count > 5 | eval alert="BRUTE_FORCE_DETECTED" | table _time, src_ip, user, count, alert # Detect lateral movement — new admin logins from unexpected hosts index=windows EventCode=4624 LogonType=3 | stats dc(ComputerName) as host_count by user | where host_count > 3
