Vulnerability Assessment Report

Status: Planned Vulnerability Management Risk Scoring Tools: OpenVAS · Nessus · CVE Database
This project is planned and will be conducted on test lab systems only. All scanning will be performed on owned/authorised machines.
Overview

Vulnerability assessment is a core skill for any security engineer. It involves systematically scanning systems to identify known weaknesses — misconfigurations, unpatched software, weak credentials — before attackers do.

This project uses OpenVAS (free, open-source) and Nessus Essentials (free for home use up to 16 IPs) to scan the home lab VMs, produce a CVSS-scored findings report, and demonstrate a complete remediation workflow.

Step-by-step methodology
01
Install and configure OpenVAS / Nessus
Set up Greenbone OpenVAS on Kali Linux or install Nessus Essentials on a Windows VM. Update the plugin feed to get the latest vulnerability signatures.
02
Define the scan scope
Target the home lab VMs (Metasploitable, Windows test VM). Document the scope formally — even for a personal lab, treating it like a real engagement builds good habits.
03
Run a credentialed and uncredentialed scan
First scan without credentials (simulates external attacker). Then scan with credentials (simulates internal threat or compromised account) to find deeper misconfigurations.
04
Prioritise findings by CVSS score
Export results. Sort by CVSS 3.1 score. Group into Critical / High / Medium / Low. Focus remediation effort on Critical and High first — this is standard enterprise practice.
05
Apply fixes and re-scan
Patch each vulnerability, apply configuration fixes, and rotate credentials. Re-run the scan to confirm each finding is resolved — this is the "verify" stage of the remediation cycle.
06
Produce the formal report
Write a professional vulnerability assessment report including: executive summary, methodology, findings table with risk ratings, remediation timeline, and post-fix verification results.
