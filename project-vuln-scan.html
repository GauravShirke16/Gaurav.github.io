01 — Overview

Project Overview & Objectives

Vulnerability assessment is one of the most fundamental skills in security engineering. It involves systematically scanning systems to identify known weaknesses — unpatched software, dangerous misconfigurations, exposed services, default credentials — before real attackers find and exploit them.

This project scans the home lab VMs (Metasploitable 2 + Windows test VM) using OpenVAS and Nessus Essentials, produces a CVSS 3.1–scored findings report, applies fixes, and runs a verification re-scan to demonstrate measurable risk reduction — the full vulnerability management lifecycle.

02 — Key Concepts

What Is Vulnerability Assessment? Key Concepts

A vulnerability assessment is a systematic, non-exploitative process of identifying known security weaknesses in a system. Unlike penetration testing, the goal is not to exploit vulnerabilities — it is to find, document, and prioritise them for remediation.

CVSS 3.1 — The Scoring Standard

CVSS (Common Vulnerability Scoring System) is the industry-standard method for rating vulnerability severity. Every CVE in the NVD has a CVSS score. Understanding it is essential for prioritising remediation effort.

CVSS ScoreSeverityTypical MeaningRemediation Priority
9.0 – 10.0CriticalRemote code execution, unauthenticated — exploit exists in the wildPatch within 24–72 hours
7.0 – 8.9HighSignificant impact — privilege escalation, data exposurePatch within 7 days
4.0 – 6.9MediumRequires local access or specific conditions to exploitPatch within 30 days
0.1 – 3.9LowMinimal impact — information disclosure, requires unusual conditionsAddress in next maintenance window
Vulnerability Assessment vs Penetration Testing
A vulnerability assessment finds and documents weaknesses — it does not exploit them. A penetration test exploits those weaknesses to prove real-world impact. This project is a vulnerability assessment. Project 01 (Home Lab) is the companion penetration test against the same targets.

04 — OpenVAS Setup

Install & Configure OpenVAS on Kali Linux

1
Install OpenVAS via apt
On Kali Linux, OpenVAS is available in the default repos:
sudo apt update && sudo apt install openvas -y
2
Run the setup script
sudo gvm-setup
This downloads ~40,000 vulnerability tests (NVTs), creates the PostgreSQL database, and generates the admin password. Takes 15–30 minutes. Copy the generated admin password.
3
Start the GVM services
sudo gvm-start
Wait for all services to start (gvmd, ospd-openvas, gsad). The web interface becomes available at https://127.0.0.1:9392
4
Login to Greenbone Web Interface
Open browser → https://127.0.0.1:9392 → accept self-signed cert → login with admin and the setup-generated password.
5
Update the NVT Feed
Administration → Feed Status → confirm all three feeds (NVT, SCAP, CERT) show "Current". If not, click Update and wait 10–20 minutes.
Expected Result
Greenbone Security Assistant loads. Feed Status shows all feeds as "Current" with 80,000+ NVTs available. The Scans dashboard shows zero scans (new installation). ✅
# Verify all GVM services are running
sudo gvm-check-setup

# Check OpenVAS version and NVT count
sudo openvas --version
sudo openvas --get-plugins | wc -l

# Reset admin password if needed
sudo gvmd --user=admin --new-password=NewSecurePass123!
05 — Nessus Setup

Install & Configure Nessus Essentials on Windows

1
Register for Nessus Essentials (Free)
Go to https://www.tenable.com/products/nessus/nessus-essentials → register with your email → receive activation code by email. Free tier supports scanning up to 16 IP addresses.
2
Download Nessus for Windows
Select Windows — AMD64 → download Nessus-10.x.x-x64.msi (~200 MB).
3
Install and activate
Run the MSI installer → accept defaults → Nessus opens automatically at https://localhost:8834. Select Nessus Essentials → enter your activation code → create admin username and password.
4
Wait for plugin compilation
Nessus downloads and compiles plugins after first activation — takes 15–45 minutes. Do not interrupt this. Status shows "Initializing" in the top bar.
5
Verify Nessus is ready
The top bar shows "Nessus Essentials" in green when ready. Navigate to Scans → My Scans — page should be empty (no scans yet).
Expected Result
Nessus Essentials web interface loads at https://localhost:8834. Plugin count shows 80,000+ plugins compiled. Ready to create scans. ✅

07 — Unauthenticated Scan

Run Unauthenticated Scan (External Attacker View)

The first scan runs without credentials — simulating what an external attacker can discover just from network-level probing. This shows your exposed attack surface from the outside.

1
Start the scan task in OpenVAS
Scans → Tasks → click the green ▶ Play button on your task. The status changes to "Requested" then "Running". A progress bar shows 0–100%.
2
Monitor scan progress
Click the task name to see live progress. The scan connects to each port, tests each service, and cross-references against the NVT database. Takes 20–60 minutes for 2 hosts.
3
Wait for "Done" status
When the task shows Status: Done, click the report link to view findings. Note the report ID for later reference.

10 — Remediation & Verification

Apply Fixes & Re-Scan (Verification)

After applying remediation, run a verification re-scan to confirm each vulnerability is resolved. This is the "Close" stage of the vulnerability management lifecycle — and the evidence that proves your work had real impact.

10.1 Remediation Commands — Metasploitable 2 (Linux)

# On Metasploitable 2 — log in as msfadmin
# Fix 1: Stop vsftpd backdoor service
sudo service vsftpd stop
sudo update-rc.d vsftpd disable

# Fix 2: Block the backdoor port used by vsftpd exploit
sudo iptables -A INPUT -p tcp --dport 6200 -j DROP
sudo iptables-save

# Fix 3: Disable Telnet service
sudo service telnetd stop
sudo update-rc.d telnetd disable

# Fix 4: Disable Samba (if not needed in lab)
sudo service smbd stop
sudo service nmbd stop
sudo update-rc.d smbd disable

# Fix 5: Change default credentials
sudo passwd msfadmin
# Enter a strong new password

# Fix 6: Disable direct root login via SSH
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sudo service ssh restart

10.2 Remediation Commands — Windows Server VM

# In PowerShell (Administrator) on Windows Server VM

# Fix 1: Disable SMBv1 (EternalBlue / WannaCry prevention)
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Get-SmbServerConfiguration | Select EnableSMB1Protocol

# Fix 2: Fix unquoted service paths (example for 2 services)
$regPath = "HKLM:\SYSTEM\CurrentControlSet\Services\VulnerableService"
Set-ItemProperty -Path $regPath -Name ImagePath -Value '"C:\Program Files\Service\service.exe"'

# Fix 3: Enable password complexity policy
secedit /export /cfg C:\secconfig.cfg
# Edit C:\secconfig.cfg: PasswordComplexity = 1, MinimumPasswordLength = 12
secedit /configure /db secedit.sdb /cfg C:\secconfig.cfg /areas SECURITYPOLICY

# Fix 4: Apply Windows Updates (run in elevated PowerShell)
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-WindowsUpdate -Install -AcceptAll -AutoReboot
