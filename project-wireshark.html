Wireshark Network Traffic Analysis

Status: Planned Network Forensics Packet Analysis Threat Hunting Tools: Wireshark · tshark · tcpdump
This project is planned. All packet captures will be taken from the isolated home lab network only — no real production traffic.
Overview

Wireshark is the world's most-used network protocol analyser. Security engineers use it to inspect raw network traffic, identify attack patterns, and conduct post-incident forensics. Every packet that travels across a network leaves evidence — Wireshark reads that evidence.

This project captures real attack traffic from the home lab (Kali Linux attacking Metasploitable), analyses the packets, and produces a threat hunting report documenting Indicators of Compromise (IOCs), command-and-control patterns, and credential transmission risks.

Step-by-step methodology
01
Capture baseline traffic
Start a Wireshark capture on the lab network interface during normal activity. Identify what "normal" looks like — DNS queries, DHCP, ARP, HTTP. Save as baseline.pcap.
02
Capture attack traffic
While running an Nmap scan and a Metasploit exploit from Kali against Metasploitable, capture all traffic. Save as attack.pcap. This gives real attack telemetry to analyse.
03
Analyse with display filters
Use Wireshark display filters to isolate suspicious traffic: port scans (SYN floods), FTP plaintext credentials, unencrypted HTTP POST bodies containing sensitive data.
04
Identify IOCs
Extract Indicators of Compromise: suspicious source IPs, unusual port usage, large outbound data transfers, C2 beacon patterns (regular intervals of outbound traffic).
05
tshark command-line analysis
Use tshark (CLI Wireshark) to automate extraction of specific fields — top talkers, DNS queries, HTTP user agents — from large PCAP files. Write Bash scripts to process PCAPs.
06
Write the forensics report
Produce a full network forensics report: timeline of events, IOCs identified, attack techniques mapped to MITRE ATT&CK framework, and recommended detection rules for a SIEM.
Useful display filters
# Show only TCP SYN packets (port scan indicator) tcp.flags.syn == 1 and tcp.flags.ack == 0 # Find FTP credentials transmitted in cleartext ftp.request.command == "PASS" # Find HTTP POST requests (possible data exfiltration) http.request.method == "POST" # Show all DNS queries (spot unusual domains) dns.flags.response == 0 # Detect ARP spoofing (duplicate IP, different MACs) arp.duplicate-address-detected
